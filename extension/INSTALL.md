# DNSentinel Browser Extension Installation Guide

This guide will walk you through setting up the DNSentinel Manifest V3 extension, including the Native Messaging Host for ML inference.

## Prerequisites
- Node.js (v18+)
- Python (3.13)
- Chrome/Edge/Brave or Firefox (developer edition for unsigned extensions)

## Step 1: Build the Extension
1. Open a terminal in the `extension/` directory.
2. Run `npm install` to install dependencies.
3. Run `npm run build` to compile the React popup, devtools, and background scripts.
   - The compiled extension will be output to the `dist/` directory.

## Step 2: Load the Extension in Chrome
1. Open Chrome and navigate to `chrome://extensions/`.
2. Enable **Developer Mode** (toggle in the top right).
3. Click **Load unpacked** and select the `extension/dist/` directory.
4. **Important:** Note the Extension ID generated by Chrome (e.g., `abcdefghijklmnop...`).

## Step 3: Install the Native Messaging Host
The extension uses a Python bridge to run the Random Forest and Isolation Forest models without needing a local web server.

1. Open `extension/native-host/com.dnssentinel.host.json`.
2. Replace `"chrome-extension://YOUR_EXTENSION_ID_HERE/"` with your actual Extension ID from Step 2.
3. Run the installation script for your OS:
   - **Windows:** Open PowerShell as Administrator and run `.\install.ps1`.
   - **Linux/macOS:** Run `chmod +x install.sh && ./install.sh`.

## Step 4: Verify Connection
1. Click the DNSentinel extension icon in your browser toolbar.
2. The popup should open. Look at the bottom footer.
3. If the Native Host is connected, you will see a green pulsing dot with "Engine Active".
4. If it fails, you will see a yellow warning: "Running in lite mode (no ML)". The extension will use fallback JS heuristics until the native host is available.

## Accessing the Forensic Dashboard
- Open Chrome DevTools (`F12` or `Ctrl+Shift+I`).
- Click the new **DNSentinel** tab to access the full SOC dashboard.
